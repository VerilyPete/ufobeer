## Problem

The codebase used `interface` for both data shapes and behavior contracts indiscriminately. 50 interfaces across 10 files were pure data shapes (request/response bodies, DB rows, config, results) that should be `type` aliases. `interface` allows accidental declaration merging and signals "behavior contract" rather than "data shape."

## Decision

Converted 50 interfaces to `type` aliases. Kept exactly 1 as `interface`: `AnalyticsEngineDataset`, which defines a `writeDataPoint()` method — a genuine behavior contract implemented by the Cloudflare runtime.

Added `readonly` to all properties during the conversion pass, including nested object shapes and `ReadonlyArray<>` for array properties.

Three types were excluded from `readonly` or needed special handling:
- `CircuitBreakerState`: properties mutated in-place — excluded (Phase 5a handles encapsulation)
- `TriggerCleanupData`: `beers_reset` was assigned post-construction — fixed by including it in the initial object literal
- `TriggerEnrichmentData`: `delete data.skip_reason` — already fixed in Phase 1 via conditional spread

Out of scope: `worker-configuration.d.ts` (auto-generated by wrangler) and `test/env.d.ts` (declaration merging for Cloudflare test env — requires `interface`).

Added a compile-time test file (`test/type-checks/interface-to-type.test.ts`) with `@ts-expect-error` annotations to verify `readonly` enforcement at the type level.

## Trade-offs

- Actual `readonly` modifier count was much higher than the ~50 conversions implied — 157 in `types.ts` alone, including nested shapes and `ReadonlyArray<>`.
- Two `@ts-expect-error` directives in the compile-check test were false positives (targeting properties that weren't actually `readonly`) — caught in review `e63fab2`.
- `FlyingSaucerBeer` index signature (`readonly [key: string]: unknown`) retained because `hasBeerStock` accesses `brewInStock` as a dynamic key. Phase 3b's schema approach handles the external trust boundary; the index signature remains for internal type safety.
- `IconSymbol.tsx`-style `as Partial<Record<...>>` widening left unchanged — `satisfies` would break the `IconSymbolName` type there.
